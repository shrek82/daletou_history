# GET /api/picks

返回全部 AI 推荐方案，包含 6 组策略推荐、2 组加权随机、5 组完全随机，以及固定号码分析。

## 请求

```
GET /api/picks
```

无请求参数。每次请求使用当前时间戳作为随机种子，生成不同的推荐结果。

| 字段 | 类型 | 说明 |
|------|------|------|
| `seed` | integer | 本次推荐使用的随机种子（Unix 时间戳秒） |
| `records_count` | integer | 用于分析的历史记录条数 |
| `strategies` | array | 6 组策略推荐方案 |
| `random` | array | 5 组完全随机方案 |
| `fixed` | object | 固定号码方案（02 22 30 33 34 + 08 12），格式与 Pick 对象一致 |

### Pick 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `index` | integer | 方案序号 |
| `red` | array[int] | 5 个红球号码 |
| `blue` | array[int] | 2 个蓝球号码 |
| `score` | float \| null | 评分，完全随机方案为 null |
| `label` | string | 策略标签 |
| `prize_stats` | object | 中奖统计 |

### prize_stats 对象

各等奖在历史记录中的命中次数：

| 字段 | 说明 |
|------|------|
| `1` | 一等奖（5+2）命中次数 |
| `2` | 二等奖（5+1）命中次数 |
| `3` | 三等奖（5+0）命中次数 |
| `4` | 四等奖（4+2）命中次数 |
| `5` | 五等奖（4+1）命中次数 |
| `6` | 六等奖（4+0）命中次数 |
| `7` | 七等奖（3+2）命中次数 |
| `8` | 八等奖（2+2 或 3+1）命中次数 |

## 策略列表

### strategies（6 组策略推荐）

| 序号 | label | 说明 |
|------|-------|------|
| 1 | 纯热号 | 选取近期出现频率最高的号码 |
| 2 | 冷热混合 | 热号为主，搭配少量遗漏号码 |
| 3 | 区间均衡 | 三个区间按比例选取 |
| 4 | 和值约束 | 选取和值在历史常见范围内的组合 |
| 5 | 同尾约束 | 包含同尾号组合 |
| 6 | 连号策略 | 包含连号组合 |

### random（5 组完全随机）

| 序号 | label | 说明 |
|------|-------|------|
| 1 | 完全随机A | 基于操作系统级加密随机源 |
| 2 | 完全随机B | 同上 |
| 3 | 完全随机C | 同上 |
| 4 | 完全随机D | 同上 |
| 5 | 完全随机E | 同上 |

## 示例

```bash
curl http://localhost:8888/api/picks
```
